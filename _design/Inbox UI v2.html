Behavior spec

Pinned row

  • Long-press any row in the list → action sheet with Pin. Long-press a pinned tile → Unpin.
  • Max 4 pinned (iMessage parity). Try to pin a 5th → "Unpin one first" toast.
  • Pinned tile is the partner's photo as a 64pt rounded-square (14pt radius) — same shape language as the inbox row avatars, just larger.
  • An amber dot top-right indicates an unread message on the pinned chat (iMessage-parity).
  • If the pinned chat has a fresh incoming message, the latest line floats over the tile as a small bubble for ~3 seconds after the message lands (then fades).
  • Pinned tiles don't show countdowns — pinned implies "I care about this one, surface it" — the urgency UX lives in the flat list.

Flat list (no day groupings)

  • One single list under a All chats label. Sorted by last_message_at desc, with one exception: unanswered incoming reach-outs (status = "needs your reply") are pinned to the top of the list regardless of recency.
  • Last-message preview is always shown. Prefix "You:" when the latest message is mine, or speaker name when it's the partner's most recent (so the user can see at a glance who owes a reply).
  • Status pins on the right edge of the meta row: Reply (dark), Hired (green), Passed (muted). Replace the older "Active" badge — implicit via meta and last message.

Unanswered reach-out countdown

  • When the other side reaches out and we haven't responded, the conversation enters status awaiting_reply. The row gets the warm tan card surface + context line "Reached out to you" + countdown pill.
  • Countdown shows time until 72-hour auto-close: created_at + 72h - now().
  • Color tier flips:
    • ≥ 24h left → muted amber pill, neutral text
    • < 24h left → solid rouge pill (urgent)
  • Format: 2d 14h left · 21h until auto-close · 14m until auto-close — collapse units once below the next threshold.
  • Lazy evaluation per existing closure-loop pattern — the inbox query computes the remaining time at read time; no cron needed for this surface.

Auto-close behavior

  • At T+72h with no reply, the conversation flips to auto_declined. Row goes faded with the muted "Auto-closed · missed reach out" context line.
  • Both users become re-eligible to appear in each other's discovery feed. On the discovery card, a small "Missed reach out" ribbon appears (Phosphor ArrowUUpLeft icon, muted tan).
  • Re-reach-out from discovery resets the conversation status to awaiting_reply, starts a fresh 72-hour timer, and inserts a system message "Reached out again after the previous request expired" so the recipient has context.
  • Either side can also tap the auto-closed row in inbox to send a new opener — same effect.

What's intentionally absent

  • No time-of-day section headers (Today / Yesterday / This week) — you asked for one flat list. Time-context is preserved per-row via the right-aligned timestamp.
  • No filter chips — pinned + flat sort + countdown is enough triage signal. Search at top handles "find a specific person."
  • No "Active now" pill — kept as the small green dot on the avatar only (consistent with the Hearth thread pill).
